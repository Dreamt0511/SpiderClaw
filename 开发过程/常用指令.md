## 自动化修复流程测试指南

### 测试流程概述
完整的端到端测试流程包括从代码提交、CI失败、自动修复到PR创建的全链路验证。

### 前置条件
1. Webhook服务已启动：`python main.py webhook start --config config/agent-config.yaml`
2. ngrok已暴露公网端口：`ngrok http 8000`（用户已经为你手动暴露端口）
3. GitHub Webhook已配置为ngrok提供的URL + `/webhook/github`路径
4. 测试仓库：`https://github.com/Dreamt0511/AutoFix_Test_rep`，必须提交测试代码到这个仓库进行CI测试
5. 测试分支：`my_test_branch`

### 测试步骤
1. **准备测试环境**
   ```bash
   # 克隆测试仓库
   git clone https://github.com/Dreamt0511/AutoFix_Test_rep.git temp_test_repo
   cd temp_test_repo
   git checkout -b my_test_branch
   ```

2. **注入语法错误**
   创建包含语法错误的Python文件，例如：
   ```python
   def process_data(items):
       result = []
       for item in items  # 缺少冒号
           if item > 0:
               result.append(item * 2)
           else  # 缺少冒号
               result.append(item)
       return result
   ```

3. **提交并推送代码**
   ```bash
   git add test_syntax_errors.py
   git commit -m "test: 添加语法错误测试文件"
   git push origin my_test_branch
   ```

4. **创建PR触发CI**
   - 在GitHub上手动创建PR从`my_test_branch`到`main`分支
   - CI运行失败后会自动触发Webhook事件

5. **监控修复流程**
   查看Webhook服务日志，验证以下环节：
   - ✅ 成功接收`workflow_run`失败事件
   - ✅ 成功下载并解析CI日志
   - ✅ 正确识别语法错误类型和位置
   - ✅ 修复Agent生成正确的修复代码
   - ✅ 审查Agent通过修复验证
   - ✅ 测试Agent验证修复有效
   - ✅ 自动提交PR或推送修复

### 常见问题排查
1. **429 TPM超出错误**
   - 症状：所有Agent调用失败，返回RateLimitExceeded
   - 解决：等待5-10分钟让配额重置，或使用更高配额的API密钥

2. **修复Agent返回不存在的文件路径**
   - 症状：修复的文件路径为"错误脚本/01单个文件报错.py"等不存在的路径
   - 解决：确保`fix_agent.py`中的强制指令正确注入实际的文件路径

3. **KeyError: '\n    "fix_description"'**
   - 症状：提示词格式化失败
   - 解决：确保JSON输出模板中的大括号正确转义，使用`{{`和`}}`而不是`{`和`}`

4. **CI日志下载403错误**
   - 症状：无法下载GitHub Action日志
   - 解决：配置正确的GitHub token，确保有仓库读取权限

### 测试验证标准
成功的测试需要满足：
1. 系统自动识别语法错误并定位到正确的文件和行号
2. 修复Agent生成的代码能够正确解决语法错误
3. 审查Agent确认修复有效且没有引入新问题
4. 测试Agent验证修复后的代码能够正常运行
5. 修复结果正确提交到仓库
