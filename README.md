# 🕷️ SpiderClaw: 事件驱动的自动诊断与修复系统

> 基于大模型的代码自动修复平台，深度对接飞书生态，实现代码错误"发现即修复"，释放研发团队生产力

---

## 🏆 项目概述

SpiderClaw 是面向研发团队的 AI 原生自动化运维平台，通过事件驱动架构实时捕获代码异常，结合大语言模型的代码理解能力实现全自动根因定位和修复，构建"异常检测→根因分析→代码修复→审查测试→PR提交→通知归档"的完整闭环。

**当前版本支持**：Python 项目全场景自动修复，覆盖语法错误、导入错误、属性错误、类型错误等常见问题。
**核心价值主张**：将线上故障平均修复时间(MTTR)从小时级缩短到分钟级，让研发团队专注于业务创新而非救火式排障。

## 📦 快速开始

### 环境要求
- Python 3.10+
- Node.js 16+（用于安装飞书CLI）
- GitHub API Token（用于仓库操作和PR创建）
- OpenAI/Claude API Key（用于大模型调用）
- 飞书企业应用（用于消息通知和多维表格）

### 安装部署
```bash
# 1. 克隆项目
git clone https://github.com/Dreamt0511/SpiderClaw.git
cd SpiderClaw

# 2. 安装依赖
pip install -r requirements.txt
pip install -e .

# 3. 安装飞书CLI
npm install -g @larksuite/cli@latest
lark-cli login  # 完成飞书应用授权

# 4. 配置文件
cp src/config/agent-config.example.yaml src/config/agent-config.yaml
# 编辑配置文件，填入GitHub、OpenAI、飞书相关配置

# 5. 启动服务
spiderclaw
```

### 最小配置示例
```yaml
# src/config/agent-config.yaml
github:
  token: "ghp_xxxxxxxxxxxx"
  allowed_repositories: ["Dreamt0511/*"]

openai:

  enabled: true
  app_id: "cli_xxxxxxxxxxxx"
  app_secret: "xxxxxxxxxxxx"
  notify_users: ["ou_xxxxxxxxxxxx"]  # 飞书用户open_id
  base_enabled: true
  base_token: "bascnxxxxxxxxxxxx"  # 飞书多维表格token，为空时自动创建
```
