你已经在 Docker Compose 中跑着 `biz-server` + `collector.sh` + `SpiderClaw Agent`，数据流已经通了：

```
模拟器 → app.log → collector.sh → POST /webhook/log → EventBus → RepairOrchestrator
```

我补充了三个文件，让这个链路完整跑起来：

### 新增文件

| 文件                                             | 作用                       |

| `docker/biz-server/app/tests/run_ci_trigger.sh`  | 容器内一键触发所有错误场景 |

### 模拟器支持的 8 个错误场景

```
divide_by_zero          — ZeroDivisionError（除零）
empty_average           — ZeroDivisionError（空列表求平均）
negative_sqrt           — ValueError（负数平方根）
user_not_found          — KeyError（查询不存在用户）
delete_nonexistent_user — KeyError（删除不存在用户）
discount_negative_rate  — 逻辑错误（不抛异常，但结果错误）
create_duplicate_email  — 逻辑错误（重复 email 无校验）
chain_reaction          — 连锁异常（除零+空列表先后触发）
```

### 五种触发方式

**方式一：一键触发所有场景（适合快速验证）**
```bash
bash tests/test_biz_error_trigger.sh all
```
这会在容器内依次执行 6 个会崩溃的场景，最后执行 `src/main.py` 自定义测试，collector 捕获后上报给 Agent。你可以切另一个终端 `docker logs spiderclaw-agent -f` 观察修复流程。

**方式二：只运行自定义测试（适合调试自己的代码）**
```bash
bash tests/test_biz_error_trigger.sh main
```
只执行 `src/main.py`，不跑标准场景。通过 collector 的 `exec` 模式捕获错误并上报。

**方式三：持续随机触发（适合长时间测试）**
```bash
bash tests/test_biz_error_trigger.sh continuous 60
```
每 60 秒随机触发一个错误场景，模拟真实业务服务间歇性报错。

**方式四：通过环境变量在容器启动时自动运行**
```bash
# 在 docker-compose.yml 的 biz-server 环境变量中加：
SIMULATOR_MODE=continuous
SIMULATOR_INTERVAL=30
SIMULATOR_ERROR_RATE=0.3
```
重启后 biz-server 会自动循环执行业务操作，按概率触发 bug。

**方式五：运行 pytest 测试**
```bash
bash tests/test_biz_error_trigger.sh pytest
```
通过 collector 的 exec 模式运行 pytest，采集测试失败信息上报。

### 自定义代码测试

如果你想测试自己的代码，编辑测试仓库中的 `src/main.py`：

```python
# AutoFix_Test_rep/src/main.py
import logging
logger = logging.getLogger(__name__)

def my_buggy_function():
    logger.info("运行自定义测试")
    result = 1 / 0  # 制造一个错误

if __name__ == "__main__":
    my_buggy_function()
```

然后推送到 GitHub，本地拉取，再运行：

```bash
# 推送到 GitHub
cd "D:\U 盘\AutoFix_Test_rep"
git add src/main.py
git commit -m "添加自定义测试"
git push

# 本地同步代码
cd "D:\U 盘\SpiderClaw\data\repos\order-service"
git pull

# 触发测试（二选一）
bash tests/test_biz_error_trigger.sh all      # 标准场景 + main.py（带有系统预设的错误测试和main.py的测试）
bash tests/test_biz_error_trigger.sh main     # 只运行 main.py
```
